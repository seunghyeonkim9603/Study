많은 물체 처리 시 문제
특히 아래 3개 분야는 매우 많은 물체를 화면에 표현해야 하는데
개인 컴퓨터에서 프로그램에 포함된 모든 물체를 처리하기에는 부하가 발생합니다.


해결 방법 1 : 물체를 모두 그리기

그래픽 카드가 모든 물체를 화면에 투영하나
화면 밖에 있는 물체는 그려지지 않기 때문에 
그릴 필요 없는 물체의 투영 비용까지 지불 되어 프로그램의 병목점이 발생합니다.


해결 방법 2 : 경계 상자, 경계 구 등 을 이용하여 추리기

화면에 확실히 안 나올 물체들은 경계 상자, 경계 구 등을 사용하여 
그래픽 카드에 요청하지 않으나 여전히 CPU에서 모든 물체에 대해
투영 연산을 필요로 함으로 병목점이 발생합니다.


위 해결방안의 문제점
모든 물체에 대해 투영연산이 발생하여
모든 물체에 투영 연산이 발생 하지 않도록 공간 분할 트리를 통해 
연산해야 하는 물체 수를 줄일 필요가 있다.

2D) Quad tree (사분 트리)
재귀적으로 2D 공간을 분할
각 공간(노드) 가 4개 (왼쪽 위, 오른쪽 위, 왼쪽 아래, 오른쪽 아래)의 자식을 가지며
최종적으로 투영연산할 필요가 있는 포인트에 재귀적으로 접근
4/1 씩 물체를 줄여나아가 투영 해야하는 물체의 개수를 줄입니다.

3D) Oct tree
개념은 사분트리와 같으나 2D와는 달리 3D는 z축을 포함하여
각노드가 8개의 자식을 가지며 3D 프로그램에 종종 사용됩니다.
